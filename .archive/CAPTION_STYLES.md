## Technical Specifications

### Positioning Standards

| Position | Height | Use Case |
|----------|--------|----------|
| Top | 10-15% | Rarely used |
| Center | 50% | Bold statements |
| **Optimal** | **58%** | **Below subject's head** |
| Bottom | 72-85% | Traditional subtitles |

**Best Practice:** 58% height positions captions below the subject's face without cutting them off.

### Animation Parameters

| Parameter | Value | Purpose |
|-----------|-------|---------|
| FPS | 30 | Smooth animation |
| Duration | 15% of word | Quick, snappy |
| Scale Range | 0.3 ‚Üí 1.5 | Strong pop |
| Word Spacing | 30px | Readability |
| Max Chars/Line | 12 | 1-2 words |

### Memory Limits

| Style Type | Max Duration | Method |
|------------|--------------|--------|
| Animated | 15 seconds | Frame-by-frame (PNG) |
| Karaoke | Unlimited | ASS subtitles |
| Static | Unlimited | ASS subtitles |

**Why 15 seconds?**
- Frame-by-frame rendering: ~450 frames (30 FPS √ó 15s)
- Memory usage: ~200-300MB during FFmpeg overlay
- Longer clips cause out-of-memory crashes

**Recommendation:** Use Karaoke style for clips over 15 seconds.

**For Viral Content (TikTok/Reels):**

## Troubleshooting

### Issue: Captions not showing
**Solution:** Check if clip duration exceeds 15 seconds for animated styles. Use Karaoke instead.

### Issue: Text reaching screen edges
**Solution:** Already fixed! Max 12 characters per line ensures proper padding.

### Issue: API crashes during export
**Solution:** Clip is too long for animated styles. Use Karaoke or reduce clip length to under 15 seconds.

### Issue: Font too small
**Solution:** All styles updated with large fonts (46-95px). If still small, try MrBeast, Rainbow, or Bounce (95px).

### Issue: Wrong positioning
**Solution:** All animated styles now use 58% height (below subject's head). Neon uses 72% (bottom third).
